# 验证的常用正则表达式

以下是一些可用于增强表单验证的正则表达式，它们超出了现代浏览器提供的范围：

## 强密码

```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$
```

确保使用至少8个字符：

* 小写字母(a-z)
* 大写字母(A-Z)
* 数字(0-9)
* 特殊字符(@$！%*?&amp;)


## 电子邮件地址


```regex
^[a-zA-Z0-9.!#$%&'*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$
```

用户名和域名中允许使用字母、数字和特殊字符。


## 电话号码（美国格式）

```regex
^\(?([0-9]{3})\)?[-. ]([0-9]{3})[-. ]([0-9]{4})$
```

以(XXX) XXX-XXXX格式验证电话号码。



## URL

```regex
^(http|https)://.*$
```

确保以http或https开头的有效URL。



## 日期(YYYY-M-DD)

```regex
^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12][0-9]|3[01])$
```

以YYYY-MM-DD格式验证日期。


## 时间(HH：MM)

```regex
^([01][0-9]|2[0-3]):[0-5][0-9]$
```

以HH：MM（24小时格式）验证时间。


## 邮政编码（美国格式）

```regex
^\d{5}(?:[-\ ]\d{4})?$
```

使用可选连字符和4位扩展名验证5位数的美国邮政编码。


## 用户名（字母数字和下划线）

```regex
^[a-zA-Z0-9_]+$
```

允许使用字母、数字和下划线。


## 颜色十六进制代码

```regex
^#[0-9a-fA-F]{6}$
```

验证6位十六进制颜色代码。 例如，#FFFFFF。


## IP 地址

```regex
^(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})$
```

验证IPv4地址。



## 社会保险号（美国格式）

```regex
^\d{3}-\d{2}-\d{4}$
```



## 信用卡号码

```regex
^(?:4[0-9]{12}(?:[0-9]{3})?|[25][1-7][0-9]{14}$
```
