# 配置感谢消息和重定向 URL

提交后的体验会在很大程度上影响用户满意度和表单完成率。Adobe 的通用编辑器提供了全面的选项，用于配置用户提交表单后看到的内容，无论是一条个性化的感谢消息，还是策略性重定向到某个特定页面。

本文提供了有关实施感谢消息和重定向 URL 的详细指导，包括技术性考虑事项、最佳做法和用户体验指南，以最大限度地增强表单提交的有效性。

## 先决条件

在配置提交后体验之前，请确保您具有：

**技术性设置：**

- 通用编辑器访问权和适当权限
- 当前有一个在通用编辑器中创建的自适应表单
- 了解您的组织对重定向 URL 的要求

**规划考虑事项：**

- **消息策略**：定义感谢消息的语气、长度和其中包含的具体信息
- **重定向策略**：确定目标页面，确保它们针对表单完成后的体验进行了优化
- **分析集成**：计划如何跟踪用户与感谢消息或重定向目标的互动

## 配置感谢消息

感谢消息可立即确认表单提交成功，可以包含个性化内容、后续步骤或与用户提交相关的重要信息。

### 何时使用感谢消息

感谢消息在以下情况下效果最佳：

- **简单确认**：用户不必额外导航，就能获得确认
- **指导性内容**：您需要提供特定的后续步骤或重要信息
- **品牌一致性**：可以按照您组织的交流风格精心设计感谢消息
- **单页体验**：让用户停留在当前页面，保持工作流的连续性

### 实施步骤

**1. 访问表单属性**

在通用编辑器中打开您的自适应表单，然后点击工具栏中的&#x200B;**编辑表单属性**&#x200B;图标。现在会打开综合表单属性对话框。

![表单属性图标](/help/forms/assets/ue-form-properties-icon.png)

**2. 导航至感谢消息配置**

在表单属性对话框中选择&#x200B;**感谢消息**&#x200B;选项卡，访问提交后配置选项。

![通用编辑器表单属性](/help/forms/assets/ue-form-properties.png)

**3. 配置消息显示**

从可用选项中选择&#x200B;**显示消息**。

![感谢](/help/edge/docs/forms/universal-editor/assets/thankyou-ue.png)

**4. 创建您的消息内容**

在&#x200B;**消息内容**&#x200B;字段中，使用富文本编辑器撰写您的感谢消息。此编辑器支持：

- **文本格式设置**：粗体、斜体、下划线和颜色选项
- **列表**：用于组织信息的项目符号和编号列表
- **链接**：导向相关资源或后续步骤的直接链接
- **全屏编辑**：点击展开图标可获得更大的编辑工作区

### 技术性考虑事项

**消息显示行为：**

- 表单提交后，消息会立即出现在一个模态叠加中
- 内容支持 HTML 格式设置，并保持响应式设计
- 用户可以忽略消息，或为消息配置自动关闭计时器

**内容指南：**

- 消息应保持简洁，同时提供必要的信息
- 在适用的情况下包含明确的后续步骤
- 考虑包含参考编号或确认详情
- 确保格式设置方法适合移动设备

### 实施示例

    感谢您提交！
    
    我们已收到您的申请，并为您分配了参考编号 #REF-2024-001234。
    
    **接下来的步骤：**
    - 您将在 15 分钟内收到一封确认电子邮件
    - 我们的团队将在 2 个工作日内审阅您提交的内容
    - 如果需要更多信息，我们将直接与您联系
    
    **需要帮助吗？** 请通过 support@example.com 联系我们的支持团队

## 配置重定向 URL

重定向 URL 会在表单提交后自动将用户导航到特定的页面，实现与现有工作流的无缝集成，或者将用户引导至相关内容。

### 何时使用重定向 URL

重定向 URL 最适合：

- **工作流集成**：将用户引导至仪表板、帐户页面或某个流程中的后续步骤
- **内容传递**：根据表单中的回复展示相关产品、服务或信息
- **分析跟踪**：将用户引导至具有特定跟踪实施的页面
- **多步骤流程**：引导用户进入复杂工作流的下一个阶段

### 实施步骤

**1. 访问表单属性**

在通用编辑器中打开您的自适应表单，然后点击&#x200B;**编辑表单属性**&#x200B;图标，打开表单配置对话框。

![表单属性图标](/help/forms/assets/ue-form-properties-icon.png)

**2. 导航至感谢消息配置**

在表单属性对话框中选择&#x200B;**感谢消息**&#x200B;选项卡，访问重定向配置选项。

![通用编辑器表单属性](/help/forms/assets/ue-form-properties.png)

**3. 启用重定向功能**

从可用的提交后选项中选择&#x200B;**重定向到 URL**。

![重定向](/help/edge/docs/forms/universal-editor/assets/redirect-ue.png)

**4. 配置目标 URL**

在提供的字段中输入您的目标 URL。系统支持多种 URL 格式，有利于灵活实施。

### URL 配置选项

**绝对 URL**

完整的网址，包括协议和域名：

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**相对路径**

相对于您当前域的路径：

    /thank-you
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sites 页面引用**

引用 AEM Sites 实施中的其他页面：

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 技术性考虑事项

**重定向行为：**

- 成功提交表单后立即重定向
- 浏览器历史记录中包括正确的后退按钮功能的重定向
- 配置重定向时间时可以选择延迟进行

**URL 验证：**

- 系统会在允许配置之前验证 URL 格式
- 根据当前的域解析相对 URL 
- 在需要的情况下，外部 URL 需要正确的 CORS 配置

## 最佳做法和推荐

### 用户体验指南

**消息优化：**

- **清晰度第一**：确保用户立即了解自己已成功提交
- **附加值**：提供信息，帮助用户进行后续步骤
- **品牌一致**：保持组织的语调和外观风格
- **考虑移动设备**：在各种屏幕尺寸上测试消息

**重定向优化：**

- **页面优化**：确保将重定向目标为表单后访问者进行优化
- **加载性能**：验证重定向页面能快速加载，保持用户体验
- **内容相关性**：确保重定向内容与表单上下文相关

### 安全考虑事项

**URL 验证：**

- 为重定向 URL 实施适当的验证，防止恶意重定向
- 考虑允许重定向域的白名单方法
- 监控重定向模式是否发生异常活动

**内容安全性：**

- 清理感谢消息内容，防止脚本注入
- 为富文本内容实施适当的内容安全性策略
- 定期对重定向目标进行安全性审查

### 分析与跟踪

**实施考虑事项：**

- **目标跟踪**：设置感谢消息浏览量和重定向完成量的分析目标
- **用户旅程地图**：跟踪用户与提交后体验的互动情况
- **转化优化**：A/B 测试不同的感谢消息和重定向目标

**衡量策略：**

- 监控用户在取消前浏览感谢消息所花的时间
- 跟踪感谢消息中链接的点击率
- 分析重定向目标页面上的用户行为

## 验证检查点

配置提交后体验之后：

**配置验证：**

- 表单属性正确显示选定的感谢消息选项
- 消息内容在预览模式下显示正确
- 重定向 URL 格式正确且可访问
- 消息中的所有链接都正确工作

**用户体验测试：**

- 提交测试表单，验证感谢消息能正确显示
- 测试不同浏览器的重定向功能
- 验证感谢消息在移动设备上的响应情况
- 确认重定向目标能正确加载

**分析设置：**

- 正确实施了感谢消息的跟踪代码
- 配置了重定向目标跟踪
- 正确触发目标完成事件

## 后续步骤

成功配置提交后体验之后：

- **监控性能**：查看分析数据，了解用户与感谢消息或重定向页面的互动情况
- **迭代和改进**：使用用户反馈和数据洞察来完善您的提交后策略
- **扩展实施**：将成功的模式应用于组织中的其他表单
