# 使用通用编辑器创建响应式Forms

用户在各种设备上访问表单，包括台式机、平板电脑和智能手机。 设计响应式表单可确保为所有用户带来最佳体验，而不管使用的是何种设备。 本指南介绍如何使用通用编辑器针对任意屏幕大小设计、测试和优化表单。


响应式表单创建涉及两个主要活动：

- **响应式测试：**&#x200B;使用设备模拟器预览和测试各种屏幕大小的表单。
- **响应式设计：**&#x200B;选择并实施无缝适应不同设备的布局模式。

**在本指南中，您将学习如何：**

- 在台式机、平板电脑和移动设备上测试表单
- 为您的内容选择适当的布局模式
- 应用响应式设计最佳实践
- 常见响应表单问题故障诊断
- 针对移动性能优化表单

## 为什么响应式Forms很重要

**用户体验影响：**

- 超过60%的用户在移动设备上访问表单
- 糟糕的移动体验会导致67%的放弃率上升
- 响应式表单最多可将完成率提高25%

**业务优势：**

- 表单完成率更高
- 提高用户满意度
- 增强的辅助功能合规性
- 降低开发和维护成本

>[!TIP]
>
> **移动优先方法：**&#x200B;开始为移动设备进行设计，然后针对更大的屏幕进行增强。 这可确保在最受限的环境中运行核心功能。

## 第1部分：跨设备测试Forms

在不同设备上测试表单有助于您在用户遇到响应式问题之前识别并解决这些问题。 Universal Editor提供了一个模拟器模式，用于模拟各种屏幕大小和方向。

### 如何测试Forms

**步骤1：打开设备模拟器**

1. 在通用编辑器中打开表单。
2. 单击工具栏中的![模拟器图标](/help/edge/docs/forms/universal-editor/assets/emulator.png){height=2%，width=2%} **模拟器**&#x200B;图标。
3. 将会显示设备选择器菜单。

![通用编辑器响应测试接口](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**步骤2：选择要测试的设备**

- **桌面** （1200px+宽度）：默认编辑视图
- **平板电脑** （768px-1199px宽度）：Medium屏幕测试
- **移动设备** （320px-767px宽度）：小屏幕测试
- **自定义**：为特定设备指定确切的维度

**步骤3：测试设备方向**

使用&#x200B;**屏幕旋转器**&#x200B;图标对两者进行测试：

- **纵向模式**：标准移动方向
- **横向模式**：旋转的平板电脑或手机视图

### 设备测试结果

每种设备类型都会显示独特的响应行为：

| **设备类型** | **屏幕宽度** | **检查内容** | **常见问题** |
|-----------------|------------------|-------------------|-------------------|
| **桌面** | 1200像素+ | 全布局，所有功能均可见 | 过多的空白，过宽的布局 |
| **平板电脑** | 768px-1199px | 组件栈叠、导航可用性 | 尴尬的规模、接触目标问题 |
| **移动设备** | 320px-767像素 | 单列布局，缩略图导航 | 小文本，间距相近的按钮 |
| **自定义** | 用户定义 | 特定于设备的要求 | 断点边缘情况 |

### 测试工作流

新Forms的&#x200B;**：**

1. **在桌面视图中生成：**&#x200B;从完整功能开始。
2. **在平板电脑上测试：**&#x200B;检查中等屏幕的适应性。
3. **在移动设备上验证：**&#x200B;确保小屏幕上的可用性。
4. **修复响应问题：**&#x200B;根据需要调整布局。
5. **重新测试所有设备：**&#x200B;确认所有大小的修复。

现有Forms的&#x200B;**：**

1. **快速移动检查：**&#x200B;表单在手机上是否有效？
2. **确定问题区域：**&#x200B;注释布局和可用性问题。
3. **系统测试：**&#x200B;彻底测试每个设备大小。
4. **文档问题：**&#x200B;跟踪需要修复的问题。
5. **实施修复：**&#x200B;有条不紊地解决问题。

## 第2部分：选择响应式布局模式

布局模式确定表单内容如何适应不同的屏幕大小。 正确的模式可改善用户体验和移动性能。

### 布局模式概述

| **布局类型** | **最适合** | **移动性能** | **复杂性** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **面板布局** | 分类的内容，仪表板样式表单 | 良好 | 低 |
| **向导布局** | 多步骤流程，复杂的工作流 | 优秀 | 中 |
| **折叠布局** | 常见问题解答样式内容，可选部分 | 优秀 | 低 |

### 快速决策指南

**在以下情况下使用面板布局：**

- 您的内容分为不同的类别
- 用户需要同时查看多个部分
- 内容相对简单

**在以下情况下使用向导布局：**

- 该表单具有多个逻辑步骤
- 想要减轻认知负担
- 移动用户是主要受众

**在以下情况下使用折叠布局：**

- 表单包含可选内容或辅助内容
- 空间保护很重要
- 内容可以在逻辑上分组

### 面板布局

**用途：**&#x200B;将相关内容组织为可同时查看的不同视觉区域。

![面板布局示例](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**响应行为：**

- **桌面(1200px+)：**&#x200B;个面板并排或网格显示
- **平板电脑(768px-1199px)：**&#x200B;个面板按间距垂直栈叠
- **移动设备(320px-767px)：**&#x200B;带有清除分区界限的单列布局

**实施步骤：**

1. 使用[面板组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel)。
2. 在每个面板中分组相关字段。
3. 为每个部分添加清除标题。
4. 确保面板之间有足够的间距。

**最佳实践：**

- 限制在台式机上使用3至4个面板，以免用户过多。
- 为每个面板使用描述性标题。
- 对相关字段进行逻辑分组，以减少认知负担。
- 在触控设备上测试面板导航。

**示例用例：**

- **工作申请：**&#x200B;个人信息、教育、经验、参考
- **产品注册：**&#x200B;基本详细信息、技术规格、保修信息
- **调查Forms：**&#x200B;人口统计、偏好设置、反馈、联系人

### 向导布局

**用途：**&#x200B;指导用户逐步完成复杂的流程，从而减少认知负荷并提高完成率。

![向导布局示例](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**响应行为：**

- **所有设备：**&#x200B;保持单步焦点，以获得最佳移动体验。
- **步骤内容：**&#x200B;在每个步骤中调整（栈叠或并排）。
- **导航：**&#x200B;具有足够间距的触摸友好按钮。
- **进度指示器：**&#x200B;可根据屏幕大小适当缩放。

**实施步骤：**

1. 使用[向导组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard)。
2. 将复杂表单拆分为逻辑步骤（最好是3 - 7个步骤）。
3. 包括用于用户定向的进度指示器。
4. 提供清晰的导航控件（“下一步”、“返回”、“保存”）。

**移动优化：**

- 对于导航按钮，请使用大触摸目标（最小为44像素）。
- 确保步骤指示符在小屏幕上清晰可见。
- 限制每步骤的字段数以减少滚动。
- 启用自动保存以防止数据丢失。

**最佳实践：**

- 确保逻辑步骤的进度 — 每个步骤都应在上一个步骤的基础上构建。
- 使用清晰的步骤标题，以便用户了解预期。
- 在每个步骤验证输入以尽早捕获错误。
- 允许用户向后导航以复查或编辑信息。

**示例用例：**

- **保险索赔：**&#x200B;事件→证据→个人→审查
- **帐户设置：**&#x200B;基本信息→首选项→安全→确认
- **订单流程：**→送货→付款→摘要

### 折叠布局

**用途：**&#x200B;通过将内容组织到可折叠的部分以节省空间，非常适用于可选信息或辅助信息。

![折叠布局示例](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**响应行为：**

- **卓越的移动性能：**&#x200B;只显示相关内容。
- **触控优化标题：**&#x200B;易于点击和展开部分。
- **平滑动画：**&#x200B;为交互提供视觉反馈。
- **空间利用率：**&#x200B;最小化所有设备上的滚动。

**实施步骤：**

1. 使用[折叠组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion)。
2. 在每个部分中对相关可选内容进行分组。
3. 使用描述性部分标题。
4. 设置适当的默认打开/关闭状态。

**移动优势：**

- 通过折叠未使用的部分来减少滚动。
- 具有自然展开/折叠手势的触控友好交互。
- 加载速度更快 — 仅显示活动内容。
- 改进的焦点 — 用户只能看到他们需要的内容。

**最佳实践：**

- 使用清除部分标题，以便用户在展开之前了解内容中的内容。
- 按逻辑方式在每个部分中对相关内容进行分组。
- 根据需要设置重要部分以开始展开。
- 提供简要的章节预览，以帮助用户决定要展开的内容。

**示例用例：**

- **产品配置：**&#x200B;基本→高级→附件→支持
- **常见问题解答Forms：**&#x200B;帐户→账单→技术→常规
- **设置Forms：**&#x200B;隐私→通知→外观→高级

## 第3部分：响应式设计最佳实践

### 按设备类型的最佳实践

+++移动设备优化(320px-767px)

**基本实践：**

- 对所有内容使用单列布局。
- 提供触摸友好型的大按钮（至少44像素高）。
- 使用明确的后退/下一页选项简化导航。
- 最小化每个部分中的滚动。
- 自动聚焦显示键盘的第一个字段。

**特定字段的准则：**

- **文本输入：**&#x200B;全宽，具有足够的内边距。
- **下拉列表：**&#x200B;使用本机选择元素获得更好的触控体验。
- **日期选取器：**&#x200B;使用本机日期输入与移动设备兼容。
- **文件上传：**&#x200B;提供大型、清除的上传区域。

+++

+++平板电脑优化(768px-1199px)

**布局策略：**

- 对相关字段使用双列布局。
- 测试纵向和横向方向。
- 支持触摸和鼠标交互。
- 提供更大的内容区域，同时保持可读性。

+++

+++台式机优化(1200px+)

**高级功能：**

- 使用多列布局以高效利用空间。
- 为超级用户提供键盘快捷键。
- 实施交互式反馈的悬停状态。
- 提供包含详细错误消息的高级验证。

+++

## 全面的故障排除

### 布局问题

+++移动设备上的表单布局分隔符

**常见原因：**

- 不缩放的固定宽度元素
- 专为桌面优先布局设计的CSS
- 使容器溢出的图像或内容

**解决方案：**

- 确保图像和容器可缩放到屏幕大小。
- 使用具有渐进式增强功能的移动优先设计。
- 使用设备模拟器和实际设备进行测试。
- 使用灵活的大小调整而不是固定的维度。

+++

+++接触目标太小

**常见原因：**

- 小于44像素×44像素的按钮
- 交互式元素放置得过于紧密
- 自定义CSS覆盖触摸友好型默认值

**解决方案：**

- 确保所有交互元素均至少为44px×44px。
- 在按钮和链接之间添加间距。
- 测试触摸与真实手指的交互，而不仅仅是鼠标。
- 增加接触目标区域以便更轻松地点按。

+++

+++内容溢出问题

**常见原因：**

- 不换行的长文本或标签
- 固定宽度容器
- 未正确缩放的图像

**解决方案：**

- 为长内容启用文本换行。
- 使用可适当缩放的响应式图像。
- 实施适应内容的灵活布局。
- 测试各种内容长度。

+++

### 性能问题

+++移动设备加载缓慢

**常见原因：**

- 未针对移动设备优化的大图像
- JavaScript执行过多
- 同时加载的表单字段过多

**解决方案：**

- 针对不同的屏幕大小优化图像。
- 仅在需要时加载非关键内容。
- 使用技术加快移动设备加载速度。
- 最大程度地减少第三方脚本和小部件。

+++

### 测试和验证问题

+++模拟器与真实设备的区别

**常见原因：**

- 浏览器特定的呈现差异
- 触摸与鼠标交互的差异
- 网络速度变化

**解决方案：**

- 尽可能在实际设备上测试。
- 使用多个浏览器进行模拟器测试。
- 在测试期间模拟不同的网络速度。
- 与目标环境中的真实用户进行验证。

+++
