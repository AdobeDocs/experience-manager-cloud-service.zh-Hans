# 使用通用编辑器创建响应式表单

现代Web环境要求各种表单，在不断扩展的设备和屏幕大小范围内无缝地发挥作用。 从大型台式机显示器到小巧的智能手机屏幕，用户无论选择什么设备，都希望获得一致、直观的体验。 创建响应式表单不再为可选操作，而是提供专业、可访问且针对转化优化的数字体验的一项基本要求。

通用编辑器提供了全面的工具和方法，用于开发智能适应各种屏幕维度、输入方法和用户上下文的响应式表单。 本指南探讨了创建表单所必需的技术基础、实施策略和优化技术，这些表单可在所有设备上出色地执行，同时保持可用性、可访问性和视觉吸引力。

响应式表单创建包括两个主要活动：

- **响应式测试：**&#x200B;使用设备模拟器在各种屏幕尺寸上预览并测试表单。
- **响应式设计：**&#x200B;选择并实施布局模式，使其无缝适应各种设备。

**在本指南中，您将了解到如何：**

- 在桌面设备、平板电脑和移动设备上测试表单
- 为您的内容选择合适的布局模式
- 应用响应式设计最佳实践
- 解决响应式表单的常见问题
- 为提高移动设备性能优化表单

## 为什么响应式表单非常重要

**影响用户体验：**

- 超过 60% 的用户是在移动设备上访问表单
- 不佳的移动设备体验会导致 67% 以上的放弃率
- 响应式表单最多可将完成率提高 25%

**商业优势：**

- 提高表单完成率
- 提高用户满意度
- 更好地符合无障碍性要求
- 降低开发和维护成本

>[!TIP]
>
> **移动设备优先方法：**&#x200B;从为移动设备设计开始，然后再改进适应更大的屏幕。这样可以确保核心功能在最受限制的环境中也能正常工作。

## 第一部分：跨设备测试表单

在不同的设备上测试您的表单可以帮助您尽早识别并解决响应式问题，避免用户遇到这类问题。通用编辑器提供模拟器模式，用于模拟各种屏幕尺寸和方向。

### 如何测试表单

**步骤 1：打开设备模拟器**

1. 在通用编辑器中打开表单。
2. 点击工具栏中的![模拟器图标](/help/edge/docs/forms/universal-editor/assets/emulator.png){height=2%,width=2%} **模拟器**&#x200B;图标。
3. 现在会出现设备选择器菜单。

![通用编辑器响应式测试界面](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**步骤 2：选择要测试的设备**

- **桌面设备**（宽度 1200px+）：默认编辑视图
- **平板电脑**（宽度 768px–1199px）：中号屏幕测试
- **移动设备**（宽度 320px–767px）：小号屏幕测试
- **自定义**：指定特定设备的确切尺寸

**步骤 3：测试设备方向**

使用&#x200B;**屏幕旋转器**&#x200B;图标来测试两种模式：

- **纵向模式**：移动设备标准方向
- **横向模式**：旋转的平板电脑或手机视图

### 设备测试结果

每种设备类型都表现出独特的响应行为：

| **设备类型** | **屏幕宽度** | **检查内容** | **常见问题** |
|-----------------|------------------|-------------------|-------------------|
| **桌面设备** | 1200px+ | 完整布局，所有功能可见 | 过多空白，布局过宽 |
| **平板电脑** | 768px–1199px | 组件堆叠，导航可用性 | 尺寸不合适，有触摸目标问题 |
| **移动设备** | 320px–767px | 单列布局，拇指导航 | 文字较小，按钮间距很近 |
| **自定义** | 用户定义 | 设备特定的要求 | 断点边缘情况 |

### 测试工作流

**对于新表单：**

1. **在桌面设备视图中构建：**&#x200B;从完整功能设计开始。
2. **在平板电脑上测试：**&#x200B;检查中号屏幕的适配情况。
3. **在移动设备上验证：**&#x200B;确保在小号屏幕上的可用性。
4. **修复响应问题：**&#x200B;根据需要调整布局。
5. **重新测试所有设备：**&#x200B;确认修复了所有尺寸设备上的问题。

**对于现有表单：**

1. **移动设备快速检查：**&#x200B;表单可以在手机上使用吗？
2. **确定问题范围：**&#x200B;注意布局和可用性问题。
3. **系统性测试：**&#x200B;彻底测试每种设备尺寸。
4. **记录问题：**&#x200B;跟踪需要修复的问题。
5. **实施修复：**&#x200B;有条不紊地解决问题。

## 第二部分：选择响应式布局模式

布局模式决定了表单内容如何适应不同的屏幕尺寸。合适的模式可以同时改善用户体验和移动设备性能。

### 布局模式概述

| **布局类型** | **最适合** | **移动设备性能** | **复杂性** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **面板布局** | 分类的内容，仪表板样式的表单 | 良好 | 低 |
| **向导布局** | 多步骤流程，复杂的工作流 | 优秀 | 中 |
| **折叠布局** | 常见问题解答样式的内容，可选的分区 | 优秀 | 低 |

### 快速决策指南

**在以下情况下使用面板布局：**

- 您的内容分为不同的类别
- 用户需要同时查看多个分区
- 内容比较简单

**在以下情况下使用向导布局：**

- 表单有多个逻辑步骤
- 您想减少认知负荷
- 移动设备用户是主要受众

**在以下情况下使用折叠布局：**

- 表单包含可选的或辅助性内容
- 节约空间很重要
- 内容可以按逻辑分成几组

### 面板布局

面板布局将相关内容组织为多个视觉上不同的部分，从而允许用户一次查看多个部分。 此布局非常适用于具有分类信息的表单，此类表单可从在较大屏幕上的并排演示中获益。

![面板布局示例](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**响应行为**

- **桌面（1200px及更高版本）：**&#x200B;个面板并排显示，或显示在网格中以便最大程度地显示。
- **平板电脑(768px-1199px)：**&#x200B;为了保持清晰度，面板以适当的间距垂直栈叠。
- **移动设备(320px-767px)：**&#x200B;面板以单列布局呈现，分区之间具有清晰的间隔，便于导航。

**如何实施**

1. 将[面板组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel)添加到您的表单。
2. 对每个面板中的相关字段进行分组，以维护逻辑组织。
3. 为每个面板部分分配清楚的描述性标题。
4. 确保面板之间有足够的间距，以防止视觉混乱。

**最佳实践**

- 将台式机上的面板数量限制为3或4，以避免用户过多。
- 为每个面板使用简洁的描述性标题以帮助用户理解。
- 以逻辑方式组织面板中的字段，将认知负荷降至最低。
- 在触控设备上测试面板导航以确保在所有平台上的可用性。

**常见用例**

- **工作申请：**&#x200B;个人信息、教育、经验和参考分区。
- **产品注册：**&#x200B;面板，了解基本详细信息、技术规格和保修信息。
- **调查Forms：**&#x200B;人口统计、偏好设置、反馈和联系信息分组。

### 向导布局

向导布局可指导用户完成一个多步骤的过程，一次演示一个部分。 此布局对于复杂表单特别有效，因为它通过将流程分成可管理的步骤来减少认知负载并提高完成率。

![向导布局示例](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**响应行为**

- **所有设备：**&#x200B;保持单步焦点，这对移动用户是最佳选择。
- **步骤内容：**&#x200B;每个步骤都以响应方式调整，可根据屏幕大小栈叠或并排排列字段。
- **导航：**&#x200B;具有触屏友好按钮，间距适中，便于轻松交互。
- **进度指示器：**&#x200B;进度条或步骤指示器可针对不同的设备适当缩放，从而提供完成状态的清晰反馈。

**如何实施**

1. 将[向导组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard)插入窗体。
2. 将表单划分为逻辑步骤（理想情况下在3到7之间），以使每个步骤都重点突出，易于管理。
3. 添加进度指示器以帮助用户了解他们在流程中的位置。
4. 提供清晰的导航控件，如“下一步”、“返回”和“保存”按钮。

**移动优化提示**

- 为导航控件使用大型触摸目标（高度至少为44像素）以增强辅助功能。
- 确保步骤指示器在小型屏幕上可见和可读。
- 限制每个步骤的字段数，以最大程度地减少滚动并提升集中度。
- 启用自动保存功能以防止用户离开表单时数据丢失。

**最佳实践**

- 设计步骤以遵循逻辑进度，每个步骤均在前一个步骤的基础上构建。
- 为每个步骤使用清晰的描述性标题来设置用户期望。
- 验证用户在每个步骤的输入以尽早捕获错误并减少挫败感。
- 允许用户向后导航以审阅或编辑以前的信息，而不会丢失数据。

**常见用例**

- **保险索赔：**&#x200B;事件详细信息、证据提交、个人信息和审核的步骤。
- **帐户设置：**&#x200B;基本信息、首选项、安全设置和确认阶段。
- **订单流程：**&#x200B;产品选择、发运信息、付款详细信息和订单汇总的步骤。

### 折叠布局

折叠布局通过将内容组织到可折叠部分来节省空间，使其非常适合用于可选或次要信息。 此布局对于包含可以逻辑分组且不需要一次全部显示的内容的表单特别有效。

![折叠布局示例](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**响应行为**

- **移动性能：**&#x200B;仅展开相关部分，从而减少滚动需要并缩短加载时间。
- **触控优化标题：**&#x200B;分区标题易于点击和展开，支持在移动设备上使用自然手势。
- **平滑动画：**&#x200B;展开和折叠部分可提供用户交互的可视反馈。
- **空间效率：**&#x200B;折叠的部分将垂直空间最小化，使表单更容易在所有设备上导航。

**如何实施**

1. 将[折叠组件](https://experienceleague.adobe.com/zh-hans/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion)添加到您的表单。
2. 每个折叠部分中与组相关的可选或辅助内容。
3. 对每个部分使用清晰的描述性标头，以帮助用户了解其中包含的信息。
4. 根据重要性和用户需求为每个部分设置适当的默认打开或关闭状态。

**移动优势**

- 通过折叠未使用的部分来减少滚动，使用户一次只关注一个部分。
- 触控友好型交互支持自然的展开/折叠手势。
- 加载速度更快，因为只有活动内容可见。
- 改进了焦点，因为用户在任何给定时间只能看到他们所需的信息。

**最佳实践**

- 使用清除章节标题，以便用户在展开章节之前了解所需。
- 按逻辑方式在每个部分中对相关内容进行分组，以便于理解。
- 设置重要部分，以便在需要立即注意时开始展开。
- 提供简短章节预览或摘要，以帮助用户决定要展开哪些章节。

**常见用例**

- **产品配置：**&#x200B;基本选项、高级设置、附件和支持的部分。
- **常见问题解答Forms：**&#x200B;帐户、帐单、技术和一般问题的分组。
- **设置Forms：**&#x200B;隐私、通知、外观和高级选项部分。


## 第三部分：响应式设计最佳实践

### 各种设备类型的最佳实践

+++移动设备优化(320px-767px)

**布局和交互：**

- 对所有表单内容使用单列布局，以最大限度地提高可读性和易用性。
- 确保所有按钮和交互元素的高度至少为44像素，以便进行可靠的触摸交互。
- 提供清晰、简单的导航，其中的“上一步”和“下一步”按钮可见。
- 通过分解长表单，最大程度地降低每个部分滚动的需要。
- 自动聚焦第一个输入字段以提示移动键盘。

**字段准则：**

- 文本字段应跨越屏幕的全部宽度，并具有足够的触屏输入边距。
- 使用本机下拉列表/选择元素以实现最佳移动可用性。
- 实施本机日期选取器以实现一致的移动体验。
- 扩大文件上传区域并明确标记以方便访问。

+++

+++平板电脑优化(768px-1199px)

**布局和可用性：**

- 利用相关字段的双列布局以利用增加的屏幕空间。
- 测试表单在纵向和横向上的外观和可用性。
- 针对触摸和鼠标输入进行设计，确保所有控件都可轻松访问。
- 增加内容区域大小，同时保持清晰的视觉层次结构和可读性。

+++

+++台式机优化(1200px+)

**高级功能和布局：**

- 使用多列布局以有效地利用水平空间并减少垂直滚动。
- 提供键盘快捷键，以便经常执行支持超级用户的操作。
- 实施交互元素的悬停状态和可视反馈。
- 为复杂表单提供高级验证功能，其中包含清晰、详细的错误消息。

## 疑难解答

### 布局问题

+++移动设备上的表单布局分隔符

**可能的原因：**

- 固定宽度的元素，不适用于较小的屏幕
- 覆盖移动样式的桌面优先CSS
- 图像或内容溢出其容器

**如何修复：**

- 确保所有图像和容器都使用相对或基于百分比的大小。
- 从面向大屏幕的移动设备优先CSS方法和增强功能图层开始。
- 使用设备模拟器和实际设备测试表单。
- 避免使用固定尺寸；使用灵活的布局。

+++

+++接触目标太小

**可能的原因：**

- 小于44px x 44px的按钮或链接
- 交互式元素之间距离太近
- 自定义CSS减小默认触控目标大小

**如何修复：**

- 确保每个交互式元素均至少为44px x 44px。
- 在按钮、链接和其他控件之间添加足够的间距。
- 用真正的触摸设备（而不仅仅是鼠标）进行测试。
- 根据需要扩展接触目标区域以实现辅助功能。

+++

+++内容溢出问题

**可能的原因：**

- 长文本或长标签不换行
- 宽度固定的容器
- 无法响应式扩展的图像

**如何修复：**

- 为所有标签和内容启用文本换行。
- 使用随容器缩放的响应图像。
- 设计灵活的布局以适应不同的内容长度。
- 测试内容短且长，以确保适应性。

+++

### 性能问题

+++移动设备加载缓慢

**可能的原因：**

- 未优化的大图像
- JavaScript过重或过多
- 同时加载的表单字段过多

**如何修复：**

- 针对移动设备优化图像并使用适当的文件格式。
- 延迟或延迟加载非关键内容。
- 最大限度地减少使用第三方脚本和小部件。
- 简化表单字段，以仅加载所需的内容。

+++

### 测试和验证问题

+++模拟器与真实设备的区别

**可能的原因：**

- 浏览器渲染引擎的差异
- 鼠标无法准确模拟触摸交互
- 网络速度差异

**如何修复：**

- 除了模拟器之外，始终在实际设备上测试。
- 使用多种浏览器和设备进行全面测试。
- 模拟各种网络速度，找出性能瓶颈。
- 从目标受众中的真实用户收集反馈。

+++

## 响应式表单的成功量度

+++关键绩效指标

**用户体验：**

- **表单完成率：**&#x200B;目标为移动设备85%或更高。
- **完成时间：**&#x200B;移动设备用户应在桌面完成时间的20%内完成表单。
- **错误率：**&#x200B;将验证错误保持在5%以下。
