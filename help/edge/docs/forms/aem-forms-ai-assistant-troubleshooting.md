# Forms Experience Builder — 故障排除指南

>[!NOTE]
>
> Forms Experience Builder在率先采用者计划下提供。 将工作地址中的电子邮件发送至`aem-forms-ea@adobe.com`以请求访问权限。

>[!IMPORTANT]
>
> **文档可能会发生更改**：此疑难解答指南当前正在针对产品进行测试，可能会受到更新和修订。 随着Forms Experience Builder在率先采用者计划中的不断发展，问题、解决方案和调试技术可能会发生变化。

此全面的疑难解答指南可帮助您识别、诊断和解决使用Forms Experience Builder时的常见问题。 该指南按问题类别组织，包含快速修复和详细解决方案。

## 快速参考 — 常见问题

| 问题 | 快速修复 |
|-------|-----------|
| **接口未加载** | 刷新浏览器，检查Internet连接，验证早期访问权限 |
| **命令不起作用** | 尝试`/help`或使用自然语言而不是斜杠命令 |
| **@fieldName无法识别** | 检查拼写，确保字段首先存在，验证字段名称语法 |
| **文件上传失败** | 在10MB以下使用PDF/JPG/PNG，检查文件格式兼容性 |
| **表单看起来错误** | 更具体一些：“使其易于移动使用”而不是“修复布局” |
| **集成失败** | 验证API凭据和权限，检查端点可用性 |
| **表单未提交** | 检查提交操作配置和验证规则 |
| **验证错误未显示** | 验证字段验证设置和错误消息放置 |
| **移动设备布局问题** | 查看响应式设计设置和字段大小 |
| **字段未显示** | 检查条件逻辑和可见性规则 |
| **导入失败** | 验证文件格式兼容性和大小限制 |
| **性能问题** | 优化字段计数并删除不必要的验证 |
| **辅助功能问题** | 审查字段标签、ARIA属性和选项卡顺序 |

**仍需要帮助？**&#x200B;键入`/help`，后跟您的特定问题，或与系统管理员联系以获得技术帮助。

## 访问和身份验证问题

### 无法访问Forms Experience Builder

**症状：**

- Forms Experience Builder界面不可见
- “访问被拒绝”或类似错误消息
- 编辑器中缺少Forms Experience Builder图标

**解决办法：**

1. **验证早期访问计划注册**
   - 确认您已获得批准加入率先采用者计划
   - 检查您的请求是否是从您的正式工作电子邮件发送的
   - 如果访问仍被挂起，请联系`aem-forms-ea@adobe.com`

2. **检查环境设置**
   - 验证是否已为您的环境启用AEM Forms
   - 确保您使用的是受支持的浏览器(Chrome、Firefox、Safari、Edge)
   - 清除浏览器缓存和Cookie
   - 禁用可能会干扰此功能的浏览器扩展

3. **验证用户权限**
   - 确认您具有适当的用户角色和权限
   - 与系统管理员确认访问权限
   - 验证您是否使用正确的帐户登录

### 接口加载问题

**症状：**

- 空白或部分加载的界面
- 旋转未完成的加载指示器
- 浏览器控制台中的JavaScript错误

**解决办法：**

1. **浏览器疑难解答**
   - 刷新页面（Ctrl+F5或Cmd+Shift+R）
   - 尝试使用其他浏览器或无痕模式/专用模式
   - 检查浏览器更新并安装（如果可用）
   - 暂时禁用广告阻止程序和隐私扩展

2. **网络连接**
   - 验证稳定的互联网连接
   - 检查公司防火墙是否阻止所需域
   - 使用不同的网络连接进行测试（如果可能）
   - 联系IT支持人员以解决网络配置问题

3. **缓存和存储问题**
   - 清除浏览器缓存和本地存储
   - 将浏览器设置重置为默认值
   - 检查设备上的可用磁盘空间
   - 尝试从其他设备访问

## 命令和交互问题

### 斜杠命令不起作用

**症状：**

- 无法识别`/create-form`或其他斜杠命令
- 未显示自动完成建议
- 命令导致出现错误消息

**解决办法：**

1. **命令语法验证**
   - 确保命令格式正确： `/command-name description`
   - 检查命令名称中的拼写错误
   - 使用自然语言作为替代语言：“创建联系人表单”
   - 尝试`/help`以验证命令可用性

2. **上下文特定的命令**
   - 验证您是否处于正确的编辑器上下文(通用编辑器与自适应Forms编辑器)
   - 某些命令仅在特定环境中可用
   - 检查命令参考以了解上下文要求

3. **替代方法**
   - 使用自然语言而不是斜杠命令
   - 将复杂的命令分解为更小、更简单的请求
   - 尝试逐步构建表单，而不是执行单个复杂的命令

### 字段引用不起作用

**症状：**

- 无法识别`@fieldName`个引用
- 有关未知字段的错误消息
- 字段修改未正确应用

**解决办法：**

1. **字段名验证**
   - 检查字段名称的准确拼写（区分大小写）
   - 引用该字段之前，请确保该字段存在
   - 使用创建的确切字段名称，而不是显示标签
   - 验证字段命名约定（camelCase、snake_case等）

2. **字段引用语法**
   - 使用不带空格的正确`@fieldName`语法
   - 避免在字段引用中使用特殊字符
   - 检查不可见字符或格式问题
   - 尝试手动重新创建字段引用

3. **调试字段引用**
   - 首先列出所有现有字段：“显示所有当前表单字段”
   - 在规则中引用字段之前创建字段
   - 使用不带复杂字符的简单字段名称
   - 测试字段一次引用一个

## 表单创建和设计问题

### 表单未按预期创建

**症状：**

- 生成的表单缺少请求的字段
- 字段类型或布局不正确
- 窗体结构与描述不匹配

**解决办法：**

1. **改进提示特殊性**
   - 在表单描述中更加详细
   - 指定准确的字段类型和验证要求
   - 包括布局首选项和用户体验要求
   - 将复杂的表单拆分为更小的增量请求

2. **迭代开发方法**
   - 从基本窗体结构开始
   - 增量添加字段和功能
   - 在继续之前测试每个添加
   - 通过对话而非单个复杂请求进行优化

3. **更好的提示示例**

   而非：

       为客户创建表单
   
   使用：

       使用以下方式创建客户联系人表单：
        — 全名（必填文本字段）
        — 电子邮件地址（验证时必需）
        — 电话号码（可选，已格式化）
        — 消息（所需文本区域，最多500个字符）
        — 提交至电子邮件通知
   

### 布局和样式问题

**症状：**

- 在移动设备上表单似乎已损坏
- 间距或对齐方式不一致
- 字段显示不正确
- 视觉层次结构较差

**解决办法：**

1. **移动响应**
   - 请求特定于移动设备的优化：“使此表单对移动设备友好”
   - 指定响应式设计要求
   - 在实际的移动设备上测试
   - 对移动设备使用单列布局

2. **布局改进**
   - 具体了解布局要求：“用两列排列地址字段”
   - 请求特定样式：“使用专业颜色和简洁的排版规则”
   - 指定间距和对齐需要
   - 要求无障碍合规性

3. **品牌一致性**
   - 在创建表单之前准备品牌准则
   - 在请求中包含特定的颜色代码和字体
   - 在所有表单中使用一致的样式
   - 创建品牌模板以供重用

### 条件逻辑问题

**症状：**

- 规则未按预期触发
- 字段显示/隐藏不正确
- 验证逻辑不起作用
- 复杂的业务规则失败

**解决办法：**

1. **规则简化**
   - 将复杂的规则分解为更小、更简单的条件
   - 组合前分别测试每个规则
   - 使用明确的特定条件：“当@maritalStatus等于‘已婚’时显示@spouseInfo”
   - 最初避免嵌套或过于复杂的逻辑

2. **规则测试和调试**
   - 测试所有可能的用户路径和方案
   - 验证条件中的字段名称和值
   - 检查规则条件中是否区分大小写
   - 使用调试模式跟踪规则执行

3. **业务逻辑实现**
   - 在实施之前清楚地记录业务要求
   - 逐步实施规则并测试每个步骤
   - 在触发规则时提供明确的用户反馈
   - 处理边缘案例和异常案例

## 文件导入和转换问题

### PDF导入失败

**症状：**

- PDF文件未上传或处理
- 转换后的表单缺少字段或内容
- PDF转换期间的错误消息
- PDF的字段识别不佳

**解决办法：**

1. **文件格式和大小**
   - 确保PDF文件小于10MB
   - 使用高质量、基于文本的PDF（不是扫描的图像）
   - 验证PDF是否不受密码保护或加密
   - 如果文本提取失败，请尝试将PDF转换为图像格式

2. **PDF质量优化**
   - 将PDF与明确定义的表单字段一起使用
   - 确保良好的对比度和可读文本
   - 避免复杂布局或元素重叠
   - 在转换请求中提供其他上下文

3. **转换增强功能**
   - 详细描述预期表单结构
   - 指定字段类型和验证要求
   - 请求特定改进：“添加移动响应性和验证”
   - 手动查看和优化转换后的表单

### 图像和屏幕快照转换问题

**症状：**

- 来自图像的较差字段识别
- 字段类型或布局不正确
- 缺少表单元素
- 转换错误或超时

**解决办法：**

1. **图像质量要求**
   - 使用高分辨率图像（至少300 DPI）
   - 确保良好的光照和对比度
   - 避免阴影、眩光或扭曲
   - 裁剪图像以仅关注表单内容

2. **最佳图像格式**
   - 使用PNG或JPG格式以获得最佳结果
   - 避免GIF或低质量的压缩图像
   - 确保图像中的文本清晰可读
   - 根据需要尝试不同的图像方向

3. **转换指南**
   - 提供表单结构的详细说明
   - 明确指定字段类型和要求
   - 在转换期间请求特定的增强功能
   - 准备在转换后进行手动调整

## 集成和提交问题

### 表单提交失败

**症状：**

- Forms未成功提交
- 提交期间出现错误消息
- 数据未到达预期目标
- 提交期间出现超时错误

**解决办法：**

1. **提交操作配置**
   - 验证是否正确配置了提交操作
   - 检查API端点和身份验证凭据
   - 先通过简单的电子邮件提交进行测试
   - 验证数据格式要求

2. **网络和连接**
   - 检查Internet连接和网络稳定性
   - 验证防火墙设置是否允许提交表单
   - 从不同的网络连接进行测试
   - 检查公司代理或安全限制

3. **数据验证问题**
   - 确保所有必填字段均已完成
   - 验证数据格式是否与API要求匹配
   - 检查特殊字符或编码问题
   - 首先使用最小数据集进行测试

### API集成问题

**症状：**

- REST API端点未响应
- 身份验证失败
- 数据格式不匹配
- 集成超时或错误

**解决办法：**

1. **API配置验证**
   - 验证API端点URL是否正确且可访问
   - 检查身份验证凭据和权限
   - 使用Postman等工具独立测试API端点
   - 验证API是否接受正确的数据格式（JSON、XML等）

2. **数据映射问题**
   - 确保表单字段名称符合API参数要求
   - 检查可能缺少的必填字段
   - 验证数据类型兼容性（字符串、数字、日期）
   - 使用示例数据进行测试以标识映射问题

3. **错误处理和调试**
   - 为API调用启用详细的错误日志记录
   - 检查API响应代码和错误消息
   - 对临时失败实施重试逻辑
   - 在API失败时为用户提供回退选项

### 电子邮件集成问题

**症状：**

- 未发送确认电子邮件
- 电子邮件转到垃圾邮件文件夹
- 电子邮件格式不正确
- 电子邮件中缺少表单数据

**解决办法：**

1. **电子邮件配置**
   - 验证电子邮件地址的格式是否正确
   - 检查SMTP设置和身份验证
   - 首先使用简单电子邮件地址进行测试
   - 验证电子邮件服务器权限和配额

2. **电子邮件投放优化**
   - 使用正确的电子邮件标头和发件人信息
   - 避免在主题行中触发垃圾邮件字词
   - 包括适当的取消订阅机制
   - 测试发送给不同提供商的电子邮件投放

3. **内容和格式**
   - 验证电子邮件中的表单数据格式是否正确
   - 检查特殊字符或编码问题
   - 测试具有各种数据组合的电子邮件模板
   - 确保电子邮件内容可访问和读取

## 性能和加载问题

### 表单加载缓慢

**症状：**

- Forms最初需要很长时间才能加载
- 用户交互缓慢
- 表单操作期间超时
- 移动设备的性能不佳

**解决办法：**

1. **表单优化**
   - 减少字段数量和复杂性
   - 为非关键部分实施延迟加载
   - 为Web交付优化图像和资产
   - 删除不必要的验证规则或逻辑

2. **浏览器和设备优化**
   - 清除浏览器缓存和临时文件
   - 关闭不必要的浏览器选项卡和应用程序
   - 检查可用的设备内存和存储
   - 尝试使用不同的浏览器进行性能比较

3. **网络优化**
   - 测试不同的网络连接
   - 检查网络拥塞或带宽限制
   - 如果可能，请使用有线连接而不是WiFi
   - 联系IT支持人员以解决网络性能问题

### 验证性能问题

**症状：**

- 验证响应缓慢
- 延迟的错误消息显示
- 验证期间表单冻结
- 字段验证期间出现超时错误

**解决办法：**

1. **验证优化**
   - 降低实时验证的频率
   - 实施验证调用的退回
   - 简化复杂的验证规则
   - 尽可能使用客户端验证

2. **规则简化**
   - 将复杂的验证分解为较小的规则
   - 删除不必要的跨字段验证
   - 优化条件逻辑以提高性能
   - 在适当时缓存验证结果

3. **用户体验改进**
   - 为简单验证提供即时反馈
   - 对复杂规则使用渐进式验证而不是实时验证
   - 在验证过程中显示加载指示器
   - 允许用户在后台验证过程中继续

## 高级故障排除

### 调试模式和诊断

**启用调试信息**

使用这些提示可获取有关表单问题的更多详细信息：

    启用调试模式以识别表单提交和字段验证的问题
    
    分析表单错误：检查验证规则、API响应和用户输入模式
    
    显示有关表单结构和字段配置的详细信息

### 错误分析技术

**系统调试方法**

1. **隔离问题**
   - 以最少的表单配置进行测试
   - 临时删除复杂功能
   - 单独测试各个组件
   - 使用消除过程确定根本原因

2. **收集诊断信息**
   - 检查浏览器控制台中是否存在JavaScript错误
   - 审查网络请求和响应
   - 记录重现问题的具体步骤
   - 收集屏幕截图和错误消息

3. **测试环境变量**
   - 尝试不同的浏览器和设备
   - 使用不同的用户帐户和权限进行测试
   - 在不同网络环境中验证
   - 与工作表单或配置比较

### 日志分析和监控

**浏览器控制台调试**

1. 打开浏览器开发人员工具(F12)
2. 检查Console选项卡以了解JavaScript错误
3. 查看网络选项卡以了解失败的请求
4. “监视性能”选项卡用于慢速操作

**常见错误模式**

- **CORS错误**： API集成的跨源请求问题
- **身份验证失败**：凭据无效或令牌已过期
- **验证错误**：字段验证规则冲突或语法错误
- **网络超时**：网络连接缓慢或不可靠

### 上报和支持

**何时联系支持人员**

- 在尝试记录的解决方案后，问题仍然存在
- 影响多个用户的系统范围问题
- 安全或数据完整性顾虑
- 需要系统级配置的集成问题

**要提供的信息**

- 问题的详细说明和重现步骤
- 问题的屏幕截图或屏幕录像
- 浏览器和系统信息
- 错误消息和控制台日志
- 表单配置和集成详细信息

**联系方式**

- 系统管理员：对于环境和访问问题
- 技术支持：针对复杂的集成和配置问题
- 提前访问项目群： `aem-forms-ea@adobe.com`特定项目群问题

### 社区和知识共享

**解决问题的最佳实践**

- 记录解决方案以供将来参考
- 与团队成员分享成功的故障排除方法
- 为组织知识库做出贡献
- 参与用户社区和论坛

**持续改进**

- 定期审查常见问题和解决方案
- 根据新的发现更新故障排除过程
- 培训和知识共享课程
- 产品团队关于功能改进的反馈

本疑难解答指南会根据用户反馈和Forms Experience Builder新增功能不断更新。 有关最新信息和其他资源，请查看[AEM Forms文档](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/home.html?lang=zh-Hans)。
