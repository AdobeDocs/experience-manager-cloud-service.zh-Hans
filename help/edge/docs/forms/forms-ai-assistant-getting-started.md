# Forms Experience Builder 快速入门指南

>[!NOTE]
>
> Forms Experience Builder 功能可在&#x200B;**早期访问 (EA) 计划**&#x200B;中使用。如果您有兴趣，请从您的工作地址向 `aem-forms-ea@adobe.com` 发送一封电子邮件，申请访问该功能。

>[!IMPORTANT]
>
> **文档可能会发生变化**：此文档目前正在针对产品进行测试，因此可能会进行更新和修订。随着 Forms Experience Builder 在早期访问计划期间不断改进，其功能、命令和示例可能会发生变化。

此综合指南可帮助您开始使用对话式 AI 技术创建和管理表单。无论您是想要创建第一个表单的初学者，还是想要利用复杂功能的高级用户，您都可以找到详细信息和实际示例，指导您了解 Forms Experience Builder 的各种功能。

## 先决条件和设置

### &#x200B;1. 申请访问

Forms Experience Builder 目前是早期访问 (EA) 计划的一部分。要参与该计划并获得访问权限，您需要提供以下信息：

**必要的信息**

- **IMS 组织 ID**：您的 Adobe 组织标识符
- **项目群 ID**：您在 Adobe Experience Cloud 内的特定项目群标识符
- **项目详细信息**：时间线、范围和预期用例
- **官方工作电子邮件地址**：与您组织的 Adobe 帐户关联的地址

**如何获取 IMS 组织 ID 和项目群 ID**

有关查找 IMS 组织 ID 和项目群 ID 的详细步骤，请参阅：

- [Adobe Experience Cloud 组织设置指南](/help/onboarding/cloud-manager-introduction.md)
- [项目群和环境管理](/help/implementing/cloud-manager/getting-access-to-aem-in-cloud/program-types.md)

**申请访问权限**

1. 使用上述指南收集您的 IMS 组织 ID 和项目群 ID
2. 发送电子邮件至 [aem-forms-ea@adobe.com](mailto:aem-forms-ea@adobe.com) 申请访问权限
3. 请在您的申请中提供：
   - 组织名称和 IMS 组织 ID
   - 项目群 ID
   - 项目时间线和范围
   - 预期用例和业务目标

>[!IMPORTANT]
>
> **限制提供计划**：Forms Experience Builder 的访问权限需经过内部利益相关者的审批。Adobe 将根据计划容量和是否符合早期访问标准来审核您的申请。不保证一定能通过审批，并且取决于当前计划的可用性。

### &#x200B;2. 验证 Forms 已启用

在使用 Forms Experience Builder 之前，请确保[您的环境已启用 AEM Forms](/help/forms/setup-forms-cloud-service.md)。


### &#x200B;3. 设置环境


- **对于 Edge Delivery Services (EDS)：**

   - [为 Edge Delivery Services Forms 设置环境](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)

   - 在您的 Adobe Experience Manager 实例上，前往“表单” > “表单和文档”
   - [使用核心组件模板创建一个新页面](/help/forms/creating-adaptive-form-core-components.md)


## 快速入门

### 访问 Forms Experience Builder

Forms Experience Builder 在 Forms Managment UI、通用编辑器和自适应表单编辑器中提供。您可以使用以下任何一种方法访问表单：

**Forms Management UI（用于核心组件）**

1. **导航至 Forms**：前往 AEM > 表单 > 表单和文档
1. 点击工具栏中的 Forms Experience Builder 图标。它位于 UI 的左上角附近。
   ![AI 助手图标*](/help/edge/docs/forms/assets/forms-manager.gif){width="50%"}
1. 开始创建对话式表单


**自适应表单编辑器（用于核心组件）**

1. 前往 AEM > 表单 > 表单和文档
2. [使用核心组件模板创建新表单](/help/forms/creating-adaptive-form-core-components.md)
3. 打开表单进行编辑
4. 点击编辑器工具栏中的 Forms Experience Builder 图标
   ![AI 助手图标*](/help/edge/docs/forms/assets/adaptive-forms-editor.gif){width="75%"}

5. 开始创建对话式表单


**通用编辑器（用于 Edge Delivery Services Forms）**

1. 按照 [Edge Delivery Services 设置指南](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)创建您的 EDS 页面
1. 在通用编辑器中导航到您的 EDS 页面
1. 在右侧面板中查找 Forms Experience Builder 图标
1. 点击打开对话界面

### 基本命令

| 符号 | 用途 | 使用示例 |
|--------|---------|---------------|
| `/` | 快速操作和快捷方式 | `/create-form contact form`、`/help validation rules`、`/update-layout wizard` |
| `@` | 引用现有的表单字段 | `@email`、`@firstName`、`Make @phoneNumber required` |
| 纯文本 | 自然对话 | “添加一个必填的电话号码字段”，“创建电子邮件地址的验证方法” |

**具体命令示例：**

- `/create-form customer survey` - 创建一个新的客户意见调查表
- `/add-field @email validation` - 添加对现有电子邮件地址字段的验证
- `/create-rule show @spouse if @maritalStatus equals married` - 创建条件逻辑
- `/configure-submit to email support@company.com` - 设置电子邮件提交
- `/help multi-step forms` - 获取有关多步骤表单创建的帮助

### 成功小建议

- **表述具体**：“添加一个必填的电子邮件地址字段和相关验证”比“添加电子邮件地址”效果更好
- **引用现有字段**：更改表格时使用 `@fieldName`
- **请求帮助**：输入 `/help` 后提出您的问题
- **采用迭代方法**：每次只进行一项更改，以获得最佳效果


## 开始创建表单的方法

### &#x200B;1. 从自然语言提示词开始

用自然语言描述您的表单要求，Forms Experience Builder 会生成完整的表单结构：

**示例：**

- “创建一个包含个人信息、财务详细信息和文件上传功能的贷款申请表”
- “创建一个包含评分、评论和产品类别的客户反馈表”
- “我需要一个包含支付处理功能的多步骤会议报名表”

### &#x200B;2. 导入和转换

将现有的表单和文档转变为现代化的交互式体验：

**受支持的来源：**

- **PDF 表单**：上传静态 PDF，将其转换为包含验证功能的交互式数字表单。
- **屏幕快照或图像**：上传纸质表单的照片，生成可以正常工作的数字版本
- **XFA 表单**：将基于旧版 XFA 的表单转换为现代化的响应式表单

**如何导入：**

1. 点击 Forms Experience Builder 界面中的附件图标
2. 上传您的文件（PDF、图像、Figma 设计等）
3. 描述您的要求：
   - “将这个 PDF 表单转换为数字版本”
   - “创建一个符合这个屏幕快照布局的表单”
   - “从我的 Figma 设计构建此表单”

**受支持的文件类型：**

- **图像**（PNG、JPG、GIF）：表单布局、UI 模型、扫描的表单、手绘草图
- **PDF 文件**：现有表单、要求说明、文档、Acroforms、XFA 表单
- **屏幕快照**：桌面/移动应用程序屏幕快照、纸质表单照片、白板草图
- **手绘草图**：餐巾纸草图、线框、概念图（拍照）

## 最佳实践

### 表单设计

- **保持简单**：从基本字段开始，只在需要时增加复杂性，以免让用户感到不知所措
- **使用清晰的标签**：使用描述性标签清晰表明字段用途，引导用户填写表单
- **提供帮助文本**：通过上下文帮助和示例引导用户完成复杂的字段
- **彻底测试**：验证所有用户路径，以确保表单在所有场景中都能正常工作

### 用户体验

- **渐进展开方式**：根据上下文显示相关字段，以减少认知负荷，提高完成率
- **清晰的导航**：帮助用户了解他们在表单中的当前位置以及剩余的步骤
- **响应式设计**：确保表单适合所有设备和屏幕尺寸，以实现最大程度的可访问性
- **无障碍**：遵循 WCAG 指南，确保残障人士也能使用表单

### 性能

- **优化字段数**：只询问必要信息，以减少表单放弃，提高完成率
- **使用适当的验证**：提交前预防错误，提供即时反馈和指导
- **测试完成率**：通过分析和用户反馈来监控和提高表单有效性
- **定期更新**：保持表单与当前的业务需求和用户期望一致，以获得最佳性能

### 品牌一致性

- **创建品牌模板**：在开始创建表单之前，准备好具有组织的颜色、字体和样式的品牌化表单模板
- **定义样式标准**：建立一致的按钮样式、字段布局和间距指南，以便在提示词中引用
- **使用品牌资产**：准备好徽标、颜色代码和品牌指南，以便在创建表单时可以轻松引用
- **模板库**：为常见用例（联系表、注册表、反馈表）构建一个品牌化表单模板收藏集
- **样式提示词**：包含品牌特定的说明：“按钮使用公司蓝色 (#1234AB)，公司字体为 Helvetica”
