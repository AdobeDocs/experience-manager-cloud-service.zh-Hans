# AEM中的代理概述 {#overview-of-agents-in-aem}

这些代理在Adobe Experience Manager (AEM) as a Cloud Service中可用，可加快内容创建并自动编排更改。

>[!IMPORTANT]
>
>代理仅在AEM as a Cloud Service和Edge Delivery Services上可用。
>
>* 它们目前不适用于AEM 6.5，也不适用于AEM 6.5 LTS。
>* 它们在AEM内部部署和Managed Services中均不可用。

## Experience Production Agent {#experience-production-agent}

[Experience Production Agent](/help/ai-in-aem/agents/production/overview.md)可自动执行高工作量和高容量任务。 增强团队的权能，将耗时数周的手动流程转变为快速的人工智能辅助工作流，让每个体验都保持最新且一致，从而帮助企业实现目标。

## 内容优化代理 {#content-optimization-agent}

[内容优化代理](/help/ai-in-aem/agents/content-optimization/overview.md)通过应用自然语言指令来创建渠道就绪的变体，来改变用户优化和调整资产的方式。 该代理解释用户意图并执行复杂的编辑任务以自动执行任务，包括生成新演绎版、调整可视属性、更改背景或为特定数字渠道准备资产。

## 发现代理 {#discovery-agent}

[发现代理](/help/ai-in-aem/agents/discovery/overview.md)提供了AEM内容，从而提供了简化的、点击免费的发现体验。 它可以跨Assets、内容片段和自适应Forms进行智能搜索，以交付相关材料，如图像、视频、PDF文档、文章和表单模板。

## 开发代理 {#development-agent}

通过简化代码的创建、调试、部署和优化，[开发代理](/help/ai-in-aem/agents/development/overview.md)可增强技术角色（开发人员和管理员）的能力。

## 治理代理 {#governance-agent}

[治理代理](/help/ai-in-aem/agents/governance/overview.md)可跨AEM保护品牌完整性和合规性。 通过强制实施安全、法规和品牌策略，代理可确保每次交互和激活都遵守既定标准。

## 限制 {#limitations}

当前存在的限制包括：

* 代理仅可用于AEM as a Cloud Service和Edge Delivery Services。
   * 它们目前不适用于AEM 6.5，也不适用于AEM 6.5 LTS。
   * 目前它们在AEM内部部署和Managed Services中均不可用。

有关特定于代理的限制，请参阅各个页面。
